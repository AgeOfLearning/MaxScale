SessionCommandSet

UMLClass
Untitled :: Model :: SessionCommandSet

Description

Container for the session command data. Used to manage the distribution of session commands to the backend servers, the storage of session commands to replay to new connections and the management of responses to the client for session commands.

Attributes

Visibility Name Type Description
public failureSemantics none

Definition of what action to take if a failure response is received from a backend connection to one of the sessions commands it has been sent.
public responseSemantics none

Defines how repsosnes are sent to the client. E.g. Send the first reponse received, the first non-error response or the last response received.

Operations

Visibility Name Description
public appendCommand()

Add a new session command to the session.

Send command to ever backend that has a position that is equal to the count of commands prior to addition of the new command
public addBackendConnection()

Add a new backend connection to the session. This will trigger the playback of any sessions commands that have been saved for this session.
public handleResponse()

Called when a response arrives from a backend databases. Updates the SessionCommand to mark this command as responsed by the backend. May also trigger sending the response to the client. If there are other commands to be sent to this backend then this will also trigger the sending of the next command.
public removeBackendConnection()

Called when a conenction is removed from the session.

The action of removing a backend may complete the set of responses required to sent the response to the client. Hence we trigger algorithm to send responses.
public isBackendUptodate()

Return true if a particular backend connection is up to date as far as session commands. I.e. the position of that BackendConnection is the sam eas the number of sessionCommands in the SessionCommandSet and the responseAwaited flag is false.
